CLI incident analysis

Understand production incidents
from logs

Sydes turns raw logs into incident structure you can investigate.

/* quickstart */
$ pip install sydes
/* query + grep */

Ask a question and narrow the evidence with an exact pattern.

$ sydes incident analyze logs/*.log \
  --query "why are checkout requests failing?" \
  --grep "503"

→ api-gateway → db-proxy timeouts causing checkout 503s
/* multiple service logs */

Analyze failures across services in a single run.

$ sydes incident analyze logs/api/*.log logs/auth/*.log logs/db/*.log \
  --query "why are session lookups failing?"

→ cache_unavailable in auth-service causing cross-service session failures
/* timeline window */

Constrain the incident window to reconstruct what changed minute by minute.

$ sydes incident analyze logs/*.log \
  --from "2026-03-25T13:34:00" \
  --to "2026-03-25T13:35:00"

13:34:05→ latency spike (api-gateway) 13:34:12→ db connections exhausted 13:34:15→ 503 surge
/* why this exists */
Logs are raw data. Incidents are structure.
Sydes reconstructs what happened so engineers investigate systems, not text.
