CLI-first incident analysis

Understand production incidents from logs in seconds.

Sydes analyzes raw logs and generates a structured root-cause hypothesis, timeline, and evidence - without manually scanning thousands of lines.

View docs →
$ sydes incident analyze logs/*.log \
  --query "Why is checkout returning 503 errors?"

→ API Gateway timeouts to db-proxy causing 503 errors on checkout requests
/* what sydes does */
  • turns raw logs into structured events
  • groups related failure signals
  • builds a timeline of what happened
  • generates an evidence-backed root-cause hypothesis
/* why this is needed */
When incidents happen, engineers are forced to scan thousands of log lines across multiple services, hosts, and timestamps. The problem is not lack of data - it is lack of structure. Sydes reconstructs what happened from logs so failures can be investigated as evidence, not line-by-line text.
/* how it works */
  • 1. parse and interpret raw logs
  • 2. group related signals into events and clusters
  • 3. build a timeline and generate a root-cause hypothesis
